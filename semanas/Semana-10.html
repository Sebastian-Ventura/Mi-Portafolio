Aprendiendo Jakarta EE

Desarrollo empresarial con estándares Java modernos

¿Qué es Jakarta EE?

Jakarta EE (anteriormente Java EE) es una plataforma para desarrollar aplicaciones empresariales robustas, escalables y seguras en Java. Proporciona APIs para manejo de dependencias, servicios web, persistencia de datos, transacciones y más.

Componentes principales de Jakarta EE

  • Jakarta Servlet – manejo de peticiones HTTP
  • Jakarta RESTful Web Services – creación de APIs REST
  • Jakarta Persistence (JPA) – mapeo objeto-relacional
  • Jakarta CDI – inyección de dependencias
  • Jakarta Faces – desarrollo de interfaces web

Ejercicio 1: Servlet básico

Este servlet imprime "Hola desde Jakarta EE" al abrir la URL.


import jakarta.servlet.*;
import jakarta.servlet.http.*;
import java.io.*;

@WebServlet("/hola")
public class HolaServlet extends HttpServlet {
  protected void doGet(HttpServletRequest req, HttpServletResponse res) throws IOException {
    res.setContentType("text/html");
    PrintWriter out = res.getWriter();
    out.println("<h1>Hola desde Jakarta EE</h1>");
  }
}

Ejercicio 2: API REST con Jakarta RESTful Web Services

Una API simple que responde con un saludo.


import jakarta.ws.rs.GET;
import jakarta.ws.rs.Path;
import jakarta.ws.rs.Produces;
import jakarta.ws.rs.core.MediaType;

@Path("/saludo")
public class SaludoResource {

  @GET
  @Produces(MediaType.TEXT_PLAIN)
  public String saludar() {
    return "¡Hola desde Jakarta REST!";
  }
}
